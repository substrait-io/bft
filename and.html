And

Defined in functions_boolean.yaml

The boolean `and` using Kleene logic. This function behaves as follows with nulls: true and null = null null and true = null false and null = false null and false = false null and null = null In other words, in this context a null value really means "unknown", and an unknown value `and` false is always false. Behavior for 0 or 1 inputs is as follows: and() -> true and(x) -> x
