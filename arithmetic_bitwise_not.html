Bitwise_not

Defined in functions_arithmetic.yaml

Return the bitwise NOT result for one integer input.

Options ¶

Kernels ¶

  • bitwise_not(i8) -> i8 : [] (not supported by dialect)
  • bitwise_not(i16) -> i16 : [] (not supported by dialect)
  • bitwise_not(i32) -> i32 : [] (not supported by dialect)
  • bitwise_not(i64) -> i64 : [] (not supported by dialect)

Dialects ¶

Details ¶

Complementary

The bitwise not function behaves complimentary with itself, i.e. bitwise_not(bitwise_not(x)) will be equal to x, for any integer.

XOR Relationship

Bitwise_not has a relationship with the XOR function, where the XORing of a valid integer with the bit pattern of all 1s results in the bitwise_not of that integer.

Two's complement

The bitwise_not of a valid integer is equivalent to negating the number and subtracting 1.

Properties ¶

NaN propagating

If any of the inputs is NaN (and the other input is not null) then the output will be NaN

Null propagating

If any of the inputs is null then the output will be null

Stateless

The output will be the same regardless of the order of input rows.
